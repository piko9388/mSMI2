🎯 무엇을 하는 시스템인가

mSMI는 material Semiconductor & Market Intelligence의 약자로, SK hynix 소재전략팀이 매주 작성하는 "외부동향 모니터링 보고서"를 자동화하는 도구입니다. Google News RSS + arXiv 논문을 수집하고, Claude API로 번역·요약·보고서 초안을 생성합니다. 단일 HTML 파일로 구성되어 GitHub Pages에서 서버 없이 동작합니다.

🏗 4계층 데이터 수집 구조

계층범위수집 내용
Tier 1🏭 산업 (26건~)글로벌 6개 권역 (KR·CN·JP·TW·US·EU) 반도체 산업 거시 동향
Tier 2🧪 소재 (30건~)7대 소재군 (Photo·Chemical·Slurry·PGas·Precursor·Wafer·Target) 시장
Tier 3🔍 BP (107사+)156개 BP사 개별 동향 (한국·일본·미국·EU·중국 10개 핵심 ⭐ 포함)
SKH💎 SK hynix (45건~)SK hynix·삼성·MU·CXMT/YMTC·DRAM/NAND 시황
학술📚 논문 (15편)arXiv EUV PR·CMP·ALD·HBM 패키징 최신 논문 (중장기 로드맵용)

⚙️ 자동화 파이프라인

① RSS 수집 (10~20초) → ② 노이즈 필터 + 관련도 점수 계산 → ③ 우선순위 자동 태깅 (🔴高/🟡中/🟢低) → ④ Top N 자동 번역 (Haiku 4.5) → ⑤ 24시간 캐싱 → ⑥ 보고서 생성 (Sonnet 4.6) → ⑦ Markdown/PDF 다운로드.

💾 데이터 저장 위치

모든 데이터는 브라우저 localStorage에 저장됩니다. 서버 전송 없음. 24시간 후 수집 결과 캐시는 자동 만료되며, 번역 캐시는 500개 LRU 방식으로 관리됩니다. API Key도 로컬에만 저장되므로 공용 PC에서는 사용을 지양하세요.

2
빠른 시작 (5단계)
처음 사용할 때 반드시 따라야 할 순서
▼
1
⚙️ 설정 탭 → API Key 입력
Direct 모드 선택 후 Anthropic Console에서 발급한 sk-ant-... Key 입력. Cloudflare Worker를 쓰는 경우엔 CF 모드로 전환하고 URL 입력. 🧪 API 테스트 버튼으로 동작 확인.
2
⚙️ 설정 탭 → BP Mode 선택
일상 사용은 top30(빠름), 주간보고 생성 시엔 all-skh(전체 127사) 또는 all(156사 포함)로 전환. Mode별 예상 RSS 호출 수는 아래 힌트에 자동 표시됩니다.
3
⟳ Refresh 버튼 클릭
상단 프로그레스 바에서 🏭→🧪→💎→🔍→📚→🤖 순서로 진행됩니다. 평균 60~120초 소요. 수집 완료 시 상단 상태 표시등이 녹색으로 바뀝니다.
4
4개 탭 순회하며 내용 검토
각 탭에서 필터·정렬·번역 확인. BP 탭에서 ⚠️ 수집 실패 N사 패널이 나타나면 쿼리 조정 필요. 🎯 점수는 관련도, 🔴高/🟡中/🟢低은 우선순위.
5
📝 보고 버튼으로 Dual-read 보고서 생성
약 60~120초 소요 (약 450원). 생성 후 모달에서 📋 복사 / 📄 .md / 📕 PDF / 🗂 Full Export 중 선택.
💡 두 번째 사용부터는? 페이지를 열면 24시간 이내 캐시가 자동 복원되므로, 바로 보고서를 생성할 수 있습니다. 새로운 데이터가 필요할 때만 ⟳ Refresh 재실행하세요.
3
탭별 세부 기능
각 탭에서 할 수 있는 일 · 필터 · 단축키
▼

🏭 산업 탭

기능: 글로벌 반도체 산업 거시 동향. 상단 통계 카드는 권역별 기사 수 표시.

필터: Region (All/KR/CN/JP/TW/US/EU) · 기간 (24h/7일/1개월/All) · 정렬 (🎯 기술순/🕐 최신순)

각 기사 카드: 🔴高/🟡中/🟢低 우선순위 뱃지, 🎯N 관련도 점수, 권역 플래그, 출처, 시간. 영문 제목은 🌐 버튼으로 수동 번역 가능 (자동 번역되지 않은 기사).

🧪 소재 탭

기능: 소재군별 시장 동향. 7대 카테고리 (Photo·Chemical·Slurry·PGas·Precursor·Wafer·Target) 필터 제공.

특징: 각 소재군당 3개 쿼리로 수집되므로 Tier 1 대비 기사는 적지만 밀도가 높습니다. 보고서 "2. 소재 BP 동향" 섹션의 주요 근거.

🔍 BP 탭

기능: 156개 BP사 개별 카드. ⭐ 표시된 카드는 중국 핵심 BP (빨간 테두리). BP명/카테고리로 검색 가능한 검색 박스 상단 제공.

필터: 소재군 13개 · 권역 6개 (All/KR/JP/US/EU/CN/TW)

정렬 로직: ⭐중국 핵심 → 기사 수 → Top 점수 → 최신 → 가나다순. 각 카드에 📰N·🎯M 뱃지로 기사 수 · 최고 점수 표시.

실패 BP 패널: 상단 빨간 박스. 쿼리를 개선할 대상을 즉시 확인 가능.

💎 SKH 탭

기능: SK hynix 중심 + 경쟁사 + 시장 시황. Focus 필터 (All/SKH/SEC/MU/CN/MKT)로 진영별 분리 가능.

용도: 보고서 "1. 칩메이커 동향" 섹션의 핵심 데이터. HBM4/HBM4E 일정, 투자 발표, 양산 성공 등 상위 이벤트 집중.

📡 Log 탭

기능: 수집 과정 전체 로그. 🔧 Debug 자동 검증 버튼으로 23개 정합성 테스트 + 프록시 네트워크 테스트 실행 가능.

활용: "수집이 빨리 끝나고 결과 0건"인 경우 Log 탭에서 프록시 실패 원인 확인. 📋 Log 복사로 전체 로그를 클립보드에 복사해 디버그 공유 가능.

⚙️ 설정 탭

주요 블록: 자동 번역 (Top N) · 키워드 가중치 (High +5 / Mid +3 / Low +1 / Neg -3) · Claude API · 노이즈 필터 · RSS 프록시 & BP Mode · Weekly Report 스타일 · 캐시 관리.

긴급 상황 대응: ♻️ 전체 초기화 & 새로고침 버튼은 설정 Key는 유지하면서 캐시·이력만 삭제. 크롤링 이상 발생 시 가장 먼저 시도할 버튼.

🚨 자주 발생하는 문제

수집이 빠르게 끝나고 결과가 0건입니다.
프록시 장애 또는 사내 방화벽 차단입니다. ① 📡 Log 탭에서 🔍 프록시 헬스 체크 결과 확인. ② ⚙️ 설정 → ♻️ 전체 초기화 & 새로고침 클릭. ③ 프록시를 rss2json 등 다른 옵션으로 변경 후 재시도. ④ 사내망 대신 LTE/5G 네트워크에서 재시도.
보고서 본문이 중간에 잘립니다.
v1.8.6에서 max_tokens 12,000으로 확대하여 해결되었습니다. 그래도 발생하면 ⚙️ 설정 → Weekly Report 스타일을 Executive로 변경하여 분량을 줄이거나, 🗂 Full Export로 Opus에서 재생성하세요.
특정 BP 기사가 수집되지 않습니다.
🔍 BP 탭 상단의 ⚠️ 수집 실패 패널에서 쿼리를 확인하세요. 쿼리가 너무 좁거나 영문명이 부정확한 경우. BP 배열의 해당 항목 q/qKr 필드를 수정 후 배포하면 다음 수집부터 적용됩니다.
API Key를 입력했는데 자동 번역이 안 됩니다.
⚙️ 설정 → 🧪 API 테스트 실행. 401/403 에러면 Key 오류, 429면 rate limit, 네트워크 에러면 CORS 차단. Direct 모드가 실패하면 Cloudflare Worker 모드로 전환 고려하세요.
📚 학술 논문 섹션이 0편입니다.
arXiv API는 corsproxy 기반으로만 동작하며 rss2json은 지원하지 않습니다. 설정 → Primary를 corsproxy 또는 allorigins로 변경 후 재시도. 그래도 실패하면 arXiv 서버 일시 장애 가능성 (1-2시간 후 재시도).
24시간 캐시를 강제로 갱신하려면?
⚙️ 설정 → 🗑️ 결과 캐시 삭제 클릭 후 ⟳ Refresh 재실행. 또는 단순히 ⟳ Refresh를 누르면 새 데이터로 덮어씌워집니다.
PDF가 팝업 차단됩니다.
v1.8.6는 팝업 차단 시 Blob URL 새 탭으로 자동 전환됩니다. 새 탭이 열리면 Ctrl+P로 PDF 저장하세요. iOS Safari는 공유 메뉴에서 PDF 저장.
이전 v1.7 보고서 스타일과 비교하고 싶습니다.
⚙️ 설정 → Weekly Report 스타일 → Prose (v1.7 호환) 선택 후 📝 보고 버튼. 같은 데이터로 줄글 버전 생성됩니다. 📂 이력에서 두 버전 모두 보관됩니다.
